# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it **privately**.

**Do not open a public GitHub issue.**

**Contact:** `[EMAIL]`
**Subject line:** `[SECURITY] threat-intel-toolkit — [brief description]`

Please include:
- A clear description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Any suggested remediation

## Response Timeline

| Stage | Timeline |
|-------|----------|
| Acknowledgment | Within 48 hours |
| Initial assessment | Within 7 days |
| Fix or remediation plan | Within 30 days |
| Public disclosure (if applicable) | Coordinated with reporter |

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest `main` | ✅ |
| Tagged releases | ✅ |
| Forks / modifications | ❌ Not supported |

## Disclosure Policy

This project follows coordinated vulnerability disclosure principles consistent with NCSC (UK) and CISA (US) guidance. We will credit researchers who report valid vulnerabilities in the release notes and SECURITY.md, unless they prefer to remain anonymous.

## CVE Policy

For critical vulnerabilities with broad impact, we will work with MITRE to request a CVE identifier and coordinate public disclosure.
